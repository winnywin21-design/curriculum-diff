# 교육과정 비교 추출 파이프라인 — README

교육과정 PDF를 넣으면 **표준 구조로 자동 추출 → 개정 간 자동 비교**까지 나오는 재사용 틀.
현재 수학과 **중학교 2015↔2022**가 끝까지 통과된 상태로 들어 있다. 다른 개정·과목·학교급은 같은 절차에 입력만 갈아 끼우면 된다.

---

## 무엇을 하나

```
 PDF ──▶ 텍스트 ──▶ [추출 프롬프트 + 스키마]──▶ 구조화 JSON ──▶ 검증 ──▶ 자동 비교(diff) ──▶ (위젯)
 (원문)   pdftotext      LLM 1회               문서당 1개      validate.py   diff.py        시각화
```

4개 층으로 되어 있고, 각 층이 파일 하나로 박제되어 있다.

| 층 | 역할 | 파일 |
|---|---|---|
| 1. 스키마 | 모든 문서가 채울 표준 칸 + 크로스워크 + 검증 규칙 | `수학교육과정_추출_표준스키마_v1.md` |
| 2. 추출 프롬프트 | 원문 → 스키마대로 JSON 뽑는 표준 명령문 | `추출프롬프트_v1.md` |
| 3. 구조화 데이터 | 추출 결과 (문서당 1개) | `math.2015.중.json`, `math.2022.중.json` |
| 4. 비교 | 두 문서 자동 대조 결과 | `diff.math.중.2015_2022.json` |
| 보조 | 자동이 못 푸는 항목의 편집자 판단(설정) | `크로스워크_항목_중.json` |
| 도구 | 검증·비교 실행 스크립트 | `validate.py`, `diff.py` |

---

## 핵심 설계 원칙

**받아쓰기(faithful) 와 정규화(normalize)를 분리한다.**
- LLM은 원문 문구를 그대로 옮긴다(해석 최소화 → 검증 가능).
- 2015·2022를 한 줄에 세우는 일은 우리가 통제하는 **크로스워크 표**가 한다. (범주→role, 영역→canonical_area, 그리고 자동이 못 푸는 항목→항목 크로스워크)

이 원칙 덕분에 "2015엔 가치·태도가 없다" 같은 사실이 **빈 칸으로 데이터에 드러나고**, 자동 비교가 손 안 대도 유지/통합/신설을 뽑아낸다.

---

## 빠른 실행 (이미 있는 데이터로)

```bash
# 1) 추출 결과 검증 (스키마 7번 규칙)
python3 validate.py math.2022.중.json
python3 validate.py math.2015.중.json

# 2) 두 문서 자동 비교 (+ 편집자 항목 크로스워크)
python3 diff.py math.2015.중.json math.2022.중.json 크로스워크_항목_중.json diff.math.중.2015_2022.json
```

크로스워크 없이 순수 자동 비교만 보려면 세 번째 인자를 빼면 된다.

## 새 문서 추가하는 법 (다음 사람용)

### A. 같은 과목, 다른 개정 (예: 차기 개정)
1. PDF → `pdftotext 파일.pdf out.txt`
2. `추출프롬프트_v1.md`의 시스템 프롬프트 + `out.txt`를 LLM에 넣어 JSON을 받는다 → `math.<연도>.중.json` 으로 저장
3. `python3 validate.py math.<연도>.중.json` 로 점검 (위반 0이어야 함)
4. `python3 diff.py 이전.json 새.json` 으로 비교

### B. 다른 학교급 (초등 / 고등)
- 초등: 영역이 더 많다(도형·측정·규칙성 등). 스키마 6번 **영역 크로스워크에 행을 추가**한다.
- 고등: 영역이 아니라 **과목**(공통수학·대수·미적분 등) 체계다. `canonical_area` 대신 **과목 크로스워크**로 분기하고, 프롬프트의 크로스워크 ②를 과목용으로 교체한다.

### C. 다른 과목
- 스키마는 그대로 두고, 두 크로스워크(범주·영역)만 그 과목 어휘로 교체하면 파이프라인이 그대로 돈다.

### 자동 비교가 ⚠검토 필요로 뜨면
항목이 자동 매칭되지 않은 것이다. 내용을 사람이 판단해 `크로스워크_항목_중.json` 형식으로 매핑을 추가하고 diff를 다시 돌리면 닫힌다. (이번 '자료와 가능성'이 그 사례.)

## 지금 담긴 결과 — 수학 중학교 2015↔2022

- **영역 5 → 4 재편**: 수와 연산(유지) · 문자와 식+함수 → **변화와 관계(통합)** · 기하 → 도형과 측정(명칭변경) · 확률과 통계 → 자료와 가능성(명칭변경)
- **지식·이해는 거의 연속** — 4영역 모두 정합. 실질 신설은 자료와 가능성의 **상자그림 1건**(학교급 하향 추정, 확인 권장).
- **가치·태도 11개 전 영역 신설** — 2015엔 대응 칸 자체가 없음.
- **영역 머리 구조 변화** — 핵심개념·일반화된 지식 → 핵심 아이디어로 통합·격상.

### 교재 편집 관점 — 중학교에서 실제 손볼 곳
1. 전 영역 **가치·태도 사이드 요소 신설** (대응물이 없어 새로 기획)
2. **변화와 관계**: 기존 ‘문자와 식’·‘함수’ 단원 묶음과 도입 서사 재편
3. **자료와 가능성**: 상자그림 신설 반영, 공학 도구·데이터 활동 강화
4. 모든 단원 도입부를 **핵심 아이디어 진술 중심**으로 다시 쓰기
