# 수학과 교육과정 추출 표준 스키마 — v1

> 범위: **내용 체계까지** (성취기준은 v2 확장 지점으로 보류)
> 목적: 교육과정 PDF를 넣으면 LLM이 이 구조로 자동 추출 → 개정·과목이 달라도 동일 포맷으로 비교 가능

---

## 설계 원칙

추출과 정규화를 **분리**한다.

- **Faithful (받아쓰기)** — LLM은 원문에 적힌 칸 이름·문구를 그대로 옮긴다. 해석 최소화 → 검증 가능.
- **Normalize (정규화)** — 2015·2022를 한 줄에 세우는 일은 우리가 통제하는 **크로스워크 표**가 한다. LLM이 임의 해석하지 않는다.

정규화 값(`canonical_area`, `role`, `role_group`)은 추출이 아니라 크로스워크가 부여한다.

---

## 1. 문서 껍데기 (envelope)

```jsonc
{
  "schema_version": "1.0",
  "document": {
    "id": "math.2022",                            // 문서 고유 id (크로스워크가 참조)
    "subject": "수학",
    "revision_year": 2022,
    "notice_no": "교육부 고시 제2022-33호",
    "document_title": "[별책 8] 수학과 교육과정",
    "framework_type": "BIG_IDEA_THREE_CATEGORY",  // FIVE_ELEMENT | BIG_IDEA_THREE_CATEGORY
    "source_file": "_별책8__2022_개정_수학과_교육과정.pdf",
    "competencies": ["문제해결","추론","의사소통","연결","정보처리"],
    "school_levels": [ /* SchoolLevel[] */ ]
  }
}
```

`framework_type` 한 칸이 "5요소 체계냐, 핵심아이디어+3범주 체계냐"를 기록한다. 추출기는 이 값으로 채울 칸을 판단한다.

- 2015: `FIVE_ELEMENT` · competencies 6개 (문제해결·추론·창의융합·의사소통·정보처리·태도및실천)
- 2022: `BIG_IDEA_THREE_CATEGORY` · competencies 5개 (문제해결·추론·의사소통·연결·정보처리)

## 2. 학교급 (SchoolLevel)

```jsonc
{
  "level": "중",                       // 초 | 중 | 고
  "grade_bands": ["중1-3"],            // 초: 1-2 / 3-4 / 5-6,  중: 중1-3
  "areas": [ /* Area[] */ ]
}
```

---

## 3. 영역 (Area) — 내용 체계의 본체

```jsonc
{
  "id": "math.2022.중.수와연산",
  "name": "수와 연산",                 // 원문 그대로 (faithful)
  "canonical_area": "NUMBER_OPERATION", // 정규화: 영역 크로스워크가 부여
  "order": 1,

  "framing": [                         // 영역 머리 진술 (role_group = FRAMING)
    { "role": "BIG_IDEA", "text": "..." }
  ],

  "content": [                         // (학년군 × 범주) 셀
    { "grade_band": "중1-3", "source_category": "지식·이해", "role": "KNOWLEDGE", "items": ["..."] },
    { "grade_band": "중1-3", "source_category": "과정·기능", "role": "PROCESS",   "items": ["..."] },
    { "grade_band": "중1-3", "source_category": "가치·태도", "role": "VALUE",     "items": ["..."] }
  ]

  // v2 확장 지점: "achievement_standards": [ ... ]
}
```

- `framing` — 핵심 아이디어·핵심개념·일반화된 지식이 모두 여기로 들어온다(모양은 달라도 자리는 같다).
- `content` — `source_category`는 원문 표기, `role`은 통제 어휘. 학년군별 셀.
- 어떤 `role` 블록이 **없으면** 그 자체가 데이터다 (예: 2015엔 VALUE 블록이 없음 → "신설" 자동 포착).

## 4. 흡수 증명 — 수와 연산 · 중학교 (이름 같은 "유지" 영역)

```jsonc
// ── 2022 (BIG_IDEA_THREE_CATEGORY) ──
{
  "id": "math.2022.중.수와연산", "name": "수와 연산",
  "canonical_area": "NUMBER_OPERATION", "order": 1,
  "framing": [
    { "role": "BIG_IDEA",
      "text": "수는 자연수에서 정수·유리수·실수로 확장되고, 사칙계산도 함께 확장되며 연산의 성질이 일관되게 성립한다." }
  ],
  "content": [
    { "grade_band": "중1-3", "source_category": "지식·이해", "role": "KNOWLEDGE",
      "items": ["소인수분해","정수와 유리수","유리수와 순환소수","제곱근과 실수"] },
    { "grade_band": "중1-3", "source_category": "과정·기능", "role": "PROCESS",
      "items": ["정수·유리수·실수의 대소 관계 판단하기","근호를 포함한 식의 사칙계산 원리 탐구하기"] },
    { "grade_band": "중1-3", "source_category": "가치·태도", "role": "VALUE",
      "items": ["음수·무리수의 필요성 인식","수 체계의 논리적 아름다움에 대한 관심"] }
  ]
}

// ── 2015 (FIVE_ELEMENT) — 같은 영역, 다른 골격 ──
{
  "id": "math.2015.중.수와연산", "name": "수와 연산",
  "canonical_area": "NUMBER_OPERATION", "order": 1,   // ★ 같은 canonical → 한 줄 비교
  "framing": [
    { "role": "CORE_CONCEPT", "text": "수의 체계" },
    { "role": "CORE_CONCEPT", "text": "수의 연산" },
    { "role": "GENERALIZED_KNOWLEDGE",
      "text": "수는 방정식의 해의 존재를 보장하기 위해 정수, 유리수, 실수 등으로 확장된다." }
  ],
  "content": [
    { "grade_band": "중1-3", "source_category": "내용 요소", "role": "KNOWLEDGE",
      "items": ["소인수분해","정수와 유리수","유리수와 순환소수","제곱근과 실수"] },
    { "grade_band": "중1-3", "source_category": "기능", "role": "PROCESS",
      "items": ["이해하기","계산하기","설명하기","추론하기","표현하기"] }
    // ★ VALUE 블록 없음 → "2015엔 가치·태도 칸이 없다"가 데이터로 드러남
  ]
}
```

`canonical_area`가 같아 좌우로 세워지고, `KNOWLEDGE` 항목은 그대로 이어지고(연속성), `framing`은 모양이 달라도 같은 자리에 들어가며, **VALUE 유무**가 곧 신설 여부를 말해준다.

## 5. 크로스워크 ① — 범주 정규화

| 원문 표기 | role | role_group |
|---|---|---|
| 핵심 아이디어 | `BIG_IDEA` | `FRAMING` |
| 핵심개념 | `CORE_CONCEPT` | `FRAMING` |
| 일반화된 지식 | `GENERALIZED_KNOWLEDGE` | `FRAMING` |
| 내용 요소 / 지식·이해 | `KNOWLEDGE` | `KNOWLEDGE` |
| 기능 / 과정·기능 | `PROCESS` | `PROCESS` |
| 가치·태도 | `VALUE` | `VALUE` |

`role`은 원문에 충실한 통제 어휘, `role_group`은 비교용 4분류(FRAMING·KNOWLEDGE·PROCESS·VALUE). 비교 레이어는 `role_group` 기준으로 두 문서를 맞춘다.

---

## 6. 크로스워크 ② — 영역 정규화 (중학교 기준 초안)

| canonical_area | 2015 | 2022 |
|---|---|---|
| `NUMBER_OPERATION` | 수와 연산 | 수와 연산 |
| `CHANGE_RELATION` | 문자와 식 + 함수 | 변화와 관계 |
| `GEOMETRY_MEASURE` | 기하 | 도형과 측정 |
| `DATA_POSSIBILITY` | 확률과 통계 | 자료와 가능성 |

> 초등은 영역 구성이 달라(도형·측정·규칙성 등) 별도 행 추가 필요. 고등은 영역이 아닌 **과목** 단위라 별도 크로스워크로 분기.

---

## 7. 검증 규칙 (추출 결과 자가 점검)

- `framework_type=FIVE_ELEMENT` 문서에는 `VALUE`·`BIG_IDEA` 블록이 없어야 한다.
- `framework_type=BIG_IDEA_THREE_CATEGORY` 문서에는 `CORE_CONCEPT`·`GENERALIZED_KNOWLEDGE` 블록이 없어야 한다.
- 모든 `content` 셀의 `grade_band`는 해당 `SchoolLevel.grade_bands` 안에 있어야 한다.
- 모든 `role`·`role_group`·`canonical_area`는 크로스워크의 통제 어휘 값만 가진다.

## 8. 파이프라인 (예고)

```
PDF → 텍스트 추출 → [추출 프롬프트 + 이 스키마] → 구조화 JSON → 검증 → 위젯 렌더
```

- 2번 층: **표준 추출 프롬프트** (다음 단계에서 작성)
